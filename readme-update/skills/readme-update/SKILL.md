# README 文档更新助手

自动检测插件市场变更，生成和更新规范的 README 文档。

## 工作流程

### 1. 扫描插件目录

遍历插件市场根目录，检测所有插件：

```
扫描目标：
├── .claude-plugin/marketplace.json  # 市场配置
└── */                               # 各插件目录
    ├── .claude-plugin/plugin.json   # 插件配置
    ├── skills/                      # 技能目录
    ├── commands/                    # 命令目录
    ├── agents/                      # 子代理目录
    └── hooks/                       # Hooks 目录
```

### 2. 收集插件信息

对每个插件提取以下信息：

| 信息项 | 来源 | 说明 |
|-------|------|------|
| 插件名称 | plugin.json → name | 插件标识符 |
| 插件描述 | plugin.json → description | 功能简介 |
| 版本号 | plugin.json → version | 当前版本 |
| 技能列表 | skills/*/SKILL.md | 统计技能数量和名称 |
| 命令列表 | commands/*.md | 统计命令数量和名称 |
| 子代理列表 | agents/*/agent.md | 统计代理数量和名称 |
| Hooks | hooks/hooks.json | 统计 Hook 配置 |

### 3. 更新根目录 README

生成插件市场总览文档，包含：

**插件统计表格**：

```markdown
## 插件列表

| 插件名称 | 描述 | 技能 | 命令 | 代理 | Hooks | 版本 |
|---------|------|------|------|------|-------|------|
| [插件A](./插件A/) | 描述... | 2 | 1 | 0 | 0 | 1.0.0 |
| [插件B](./插件B/) | 描述... | 1 | 0 | 1 | 1 | 1.0.0 |
```

**使用说明**：
- 添加市场源的命令
- 安装插件的命令
- 本地开发说明

**目录结构**：
- 完整的目录树展示

### 4. 更新插件 README

为每个插件生成或更新 README.md，包含：

**基本信息**：

```markdown
# 插件名称

插件描述

## 安装

/plugin install 插件名@市场名

## 功能列表

### 技能 (Skills)

| 技能名称 | 描述 | 触发方式 |
|---------|------|---------|
| skill-name | 描述... | 关键词/命令 |

### 命令 (Commands)

| 命令 | 描述 |
|------|------|
| /plugin:command | 描述... |

### 子代理 (Agents)

| 代理名称 | 描述 |
|---------|------|
| agent-name | 描述... |

### Hooks

| 事件 | 匹配器 | 动作 |
|------|-------|------|
| PostToolUse | Write | 命令... |
```

**详细使用说明**：
- 每个技能的详细用法
- 示例输入输出
- 注意事项

## 输出规范

### 文件编码
- UTF-8 编码
- LF 换行符

### Markdown 格式
- 使用标准 GitHub Flavored Markdown
- 表格对齐、代码块标注语言
- 适当使用标题层级（h1-h4）

### 更新策略
- 保留用户自定义内容（通过标记注释）
- 仅更新自动生成的统计部分
- 新增插件自动追加到列表

## 触发方式

- `更新 README`
- `同步文档`
- `生成插件统计`
- `更新插件市场文档`
